# Plan 1.2: WebView の生成とリサイズ同期

## Objective
Rust 側で `WebviewBuilder` と `add_child` を使用し、実際の WebView をサイドバーの右側に表示します。また、ウィンドウサイズ変更に合わせて WebView が自動でリサイズされるようにします。

## Context
- .gsd/phases/1/RESEARCH.md
- src-tauri/src/lib.rs

## Tasks

<task type="auto">
  <name>WebView 生成ロジックの実装</name>
  <files>
    - src-tauri/src/lib.rs
  </files>
  <action>
    1. `create_webview` コマンドを実装。
    2. `tauri::webview::WebviewBuilder` を使用して新規 WebView を作成。
    3. `window.add_child` を呼び出し、x: 200, y: 0 の位置に配置（サイドバーの幅を考慮）。
    4. WebView インスタンスを Rust 側の状態（State またはウィンドウのリソース）として管理できる仕組みを検討。
    ※ 初回は google.com をデフォルトとして読み込む。
  </action>
  <verify>App.tsx のマウント時に create_webview を呼び出し、ウィンドウの右半分に Google が表示されることを確認。</verify>
  <done>アプリ起動後、サイドバーの右側に指定した URL のページが表示されること。</done>
</task>

<task type="auto">
  <name>リサイズ同期ロジックの実装</name>
  <files>
    - src-tauri/src/lib.rs
  </files>
  <action>
    1. メインウィンドウの `on_window_event` を使用。
    2. `WindowEvent::Resized` イベント発生時に、子 WebView のサイズを更新。
    3. 新しいサイズは `window_width - 200` (サイドバー分) と `window_height` に設定。
  </action>
  <verify>ウィンドウをドラッグしてリサイズした際、WebView の中身も領域いっぱいに広がるか追従することを確認。</verify>
  <done>ウィンドウリサイズ時、WebView が常にサイドバー以外の領域をすべて埋めるように自動調整されること。</done>
</task>

## Success Criteria
- [ ] Child WebView がサイドバーの右側に正しく配置されている。
- [ ] ウィンドウリサイズに合わせて WebView のサイズも自動更新される。
