# Backend Task: AdBlock Implementation (Basic)

## 概要
簡易的なドメインブロック機能を実装し、不要なコンテンツの読み込みを遮断する。
将来的な拡張を見据え、まずはリクエストインターセプトの仕組みを導入する。

## 要件
1.  **リクエストインターセプト**:
    -   Content Webviewからのリクエストを監視。
    -   ブロックリスト（ドメインリスト）と照合。
2.  **ブロックリスト管理**:
    -   静的なリスト（ハードコードまたはローカルファイル）からブロック対象ドメインを読み込む。
3.  **遮断動作**:
    -   対象ドメインへのリクエストをキャンセルする。

## 実装ステップ
1.  Tauri (または `wry`) の `on_request` などのフック機能、またはカスタムプロトコルハンドラを使用してリクエストを捕捉できるか調査・実装。
    -   *注: Tauri v2のAPIでリクエストフィルタリングがどの程度サポートされているか確認が必要。*
2.  単純なドメインマッチングロジックを実装。
3.  テスト用ドメイン（例: doubleclick.net）で遮断を確認。

## 完了条件
-   指定したドメイン（広告配信サーバ等）へのアクセスがブロックされること。
