## ベストプラクティス

実装中に発見したベストプラクティスをここに記録します。

### Rust側のエラーハンドリング

- `Result<T, String>`ではなく、`thiserror`を使ったカスタムエラー型を定義すると、エラーメッセージが一貫して分かりやすくなる
- 例:
  ```rust
  #[derive(Debug, thiserror::Error)]
  pub enum MuError {
      #[error("タブが見つかりません: {0}")]
      TabNotFound(String),
  }
  ```

### Reactコンポーネントのメモリ管理

- `useEffect`で設定したリスナーやタイマーは、必ずクリーンアップ関数で解除する
- 例:
  ```typescript
  useEffect(() => {
    const handleKeyDown = (e: KeyboardEvent) => { /* ... */ };
    window.addEventListener('keydown', handleKeyDown);
    return () => window.removeEventListener('keydown', handleKeyDown);
  }, []);
  ```

### Tailwind CSSのパフォーマンス

- ビルド時に未使用のCSSを削除するため、`tailwind.config.js`の`content`設定を正確に指定する
- カスタムクラスは最小限に抑え、Tailwindのユーティリティクラスを活用する

## トラブルシューティング

### よくある問題と解決策

#### 問題: `bun run tauri dev`でエラーが発生する

**症状**: Tauriの開発サーバーが起動しない

**確認事項**:
1. Rustがインストールされているか（`rustc --version`）
2. bunがインストールされているか（`bun --version`）
3. `src-tauri/target/`を削除してから再ビルド

**解決策**: 大抵の場合、`src-tauri/target/`を削除して再ビルドで解決

#### 問題: WebViewでページが表示されない

**症状**: URLを入力してもページが表示されない

**確認事項**:
1. URLが正しいか（http://またはhttps://で始まっているか）
2. Tauri設定でWebViewが有効になっているか
3. CORS設定が適切か

**解決策**: `tauri.conf.json`の`dangerousRemoteDomainIpcAccess`を確認
